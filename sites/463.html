UFR DE MATHéMATIQUES ET D'INFORMATIQUE - UNIVERSITé PARIS CITé

L’Unité de Formation et de Recherche (UFR) de Mathématiques et d’Informatique est l’une des dix composantes d’Université Paris Cité, elle est responsable de la formation et de la recherche en informatique et en mathématiques appliquées de l’université.

 

 

Le défi: Améliorer les fonctionnalités d'Open WebUI

Theme:

IA générative, contenu multi-canal, podcast

Lot:

Un lot d'enceintes connectées pour l'équipe ayant proposée la meilleure réalisation

Open WebUI (https://github.com/open-webui/open-webui) est un projet open-source qui vise à développer une interface web dotée de nombreuses fonctionnalités pour interagir avec des grands modèles de langage auto-hébergés.

Ce defi consiste à enrichir cette interface avec de nouvelles fonctionnalités à ce projet afin d'augmenter les possibilités offertes.

Elements attendus

Nous souhaiterions voir développer de nouvelles fonctionnalités qui s'intègre dans l'application que vous allez réaliser autour du sujet de la nuit de l'info 2024 :

  • Génération de contenus formatés et cohérents pour une communication multi-canal à partir d’une collection de documents constituée dans un espace de travail. En particulier, podcasts, articles de blogs, … Cette génération peut être envisagée à partir d'autres outils open-source ou à partir d'enchainement de prompts pour générer un contenu adapté à chaque canal. Le contenu généré doit répondre aux meilleures pratiques du canal visé. Idéalement, la génération de podcast doit intégrere la synthèse vocale et le montage. Pour chaque type de contenu, un mode interactif pour y apporter des corrections serait idéal. Pour chaque sortie, une possibilité de sauvegarde adaptée est nécessaire pour rendre la fonctionnalité exploitable.
  • Composition musicale pour faire compléter les possibilités offertes par la génération d’images (basé sur d'autres outils open-source)

Tout doit pouvoir être auto-hébergé librement et ne donc ne pas faire appel à des API commerciales.

Mode de restitution

Idéalement, nous souhaiterions que les modifications soient acceptées par la communauté du projet. A défaut, le code des modifications doit être accessible sur github ou une plateforme équivalente. Pour évaluer le résultat, une image docker (ou un lien web vers l'application) est souhaitée et en complément une installation hébergée sur un serveur serait idéale.
